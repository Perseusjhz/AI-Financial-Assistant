# 2. 项目背景

## 2.1 赛题背景

随着互联网理财普及、生活费自主支配权提升，当代在校大学生已经成为理财入门群体的重要组成部分。大学生普遍拥有日常生活费、兼职收入、奖学金等可支配资金，但在实际生活中存在以下问题：

- 缺乏长期攒钱规划。
- 缺少应急储蓄意识。
- 消费管控能力不足。
- 理财认知薄弱。
- 容易盲目跟风。
- 忽视投资风险。
- 缺少可以互相监督交流的理财搭子。

现有通用理财软件大多偏向专业投资者，信息繁杂、功能复杂，不贴合校园学生真实使用场景，也缺少轻量化、陪伴式、非营销型的理财辅助工具。

因此，本项目需要设计并搭建一款以智能对话机器人为核心形态的大学生理财陪伴 AI 智能体，落地 2–3 项核心能力，并提供可体验二维码。

## 2.2 产品机会

大学生的理财需求并不等同于成熟投资者的理财需求。

成熟投资者关注：

- 买什么产品。
- 收益如何。
- 如何配置资产。
- 如何交易。
- 如何控制资产组合风险。

大学生更真实的需求是：

- 这个月钱够不够花。
- 能不能攒下目标金额。
- 今天哪些消费不必要。
- 生活费能不能拿去理财。
- 基金、股票、虚拟资产到底有什么风险。
- 如何从零开始建立预算和储蓄习惯。

因此，本产品的机会不是“帮大学生买理财产品”，而是：

> 帮大学生建立预算、储蓄、消费复盘和风险认知等基础财务习惯。

## 2.3 项目核心判断

本项目不应设计为销售型理财工具，而应设计为：

> 面向大学生的财务习惯养成 Agent。

产品核心价值不是推荐收益，而是帮助学生：

1. 管住今天的钱。
2. 攒下未来的钱。
3. 看懂理财的风险。

# 3. 产品概述

## 3.1 产品名称

**攒钱搭子 AI**

## 3.2 产品定位

攒钱搭子 AI 是一款面向在校大学生的手机端 AI 理财陪伴智能体。产品通过智能对话、任务型 Workflow、轻量 RAG 知识库和风险边界控制，帮助大学生完成攒钱规划、消费复盘和入门理财风险教育。

## 3.3 一句话定位

> 面向大学生的轻量化、陪伴式、非营销型理财搭子，帮助学生把生活费管明白。

## 3.4 产品口号

> 不推销理财产品，只陪你把生活费管明白。

## 3.5 产品形态

| 项目 | 说明 |
|---|---|
| 终端形态 | 手机端 H5 / PWA |
| 访问方式 | 二维码扫码访问 |
| 核心入口 | AI 对话机器人 |
| 交互方式 | 场景按钮 + 自然语言输入 |
| 输出方式 | 聊天回复 + 结构化结果卡片 |
| 后端能力 | Agent Workflow + RAG + LLM API |
| 部署方式 | DigitalOcean 云服务器 |
| 展示方式 | PPT + 3 分钟横屏 Demo 视频 |

# 4. 产品目标

## 4.1 业务目标

本项目为比赛 Demo，不追求商业化转化，目标是构建一个完整、可体验、可讲清楚的 AI 产品方案。

业务目标包括：

1. 在产品层面准确命中大学生理财入门痛点。
2. 在功能层面落地 2–3 项核心能力。
3. 在技术层面体现 Agent Workflow，而非简单套壳大模型。
4. 在展示层面通过手机端 Demo、PPT 和视频形成完整闭环。
5. 在评审层面体现产品经理的需求分析、产品定位、智能体设计、功能实现、评测能力。

## 4.2 用户目标

帮助大学生完成：

1. 制定可执行的攒钱计划。
2. 复盘日常消费结构。
3. 识别冲动消费和可优化消费。
4. 理解入门理财风险。
5. 避免盲目跟风和借贷投资。
6. 建立基础预算、储蓄和风险意识。

## 4.3 产品成功标准

| 维度 | 成功标准 |
|---|---|
| 可体验性 | 用户扫码后可在手机端完成三条核心路径 |
| 可理解性 | 用户 3 秒内理解产品定位和功能入口 |
| 可用性 | 三个核心功能可稳定返回结构化结果 |
| 智能体特征 | 系统具备信息追问、Workflow、风险边界和结构化输出 |
| 安全性 | 不推荐具体金融产品，不承诺收益，不鼓励借贷投资 |
| 展示效果 | PPT 和视频能够清晰讲出产品价值、Agent 逻辑和核心功能 |
| 评测完整度 | 提供评测方式和初步评测结果 |

# 5. 目标用户与用户画像

## 5.1 目标用户范围

目标用户为在校大学生，尤其是具有以下特征的用户：

- 每月有固定生活费。
- 有兼职收入或奖学金。
- 经常月末预算紧张。
- 有攒钱目标。
- 缺少记账和复盘习惯。
- 对理财概念感兴趣但认知初级。
- 容易被高收益内容影响。
- 不喜欢专业复杂的理财工具。

## 5.2 用户画像 1：月末吃紧型学生

### 基本信息

| 字段 | 内容 |
|---|---|
| 年级 | 大一 / 大二 |
| 月生活费 | 2000–3000 元 |
| 收入结构 | 主要来自家庭生活费 |
| 使用习惯 | 不记账，靠余额感知消费 |
| 典型问题 | 月初宽松，月末吃紧 |

### 核心痛点

- 不知道每天能花多少钱。
- 奶茶、外卖、打车等高频支出难以控制。
- 缺乏消费复盘。
- 月末才发现预算超支。

### 产品价值

- 生成每日预算。
- 帮助识别可优化消费。
- 提供明日行动建议。

## 5.3 用户画像 2：目标攒钱型学生

### 基本信息

| 字段 | 内容 |
|---|---|
| 年级 | 大二 / 大三 |
| 月生活费 | 2500–4000 元 |
| 额外收入 | 兼职、奖学金、实习补贴 |
| 典型目标 | 买电脑、旅行、考试报名、课程学习 |
| 典型问题 | 有目标但无法持续执行 |

### 核心痛点

- 不知道攒钱目标是否可行。
- 不会把目标拆解到每日预算。
- 容易被临时消费打断。
- 没有阶段性检查机制。

### 产品价值

- 判断目标可行性。
- 生成月度、周度、日度预算。
- 给出本周执行建议。

## 5.4 用户画像 3：理财小白型学生

### 基本信息

| 字段 | 内容 |
|---|---|
| 年级 | 大一到研究生 |
| 理财经验 | 初级 |
| 信息来源 | 短视频、社交平台、同学推荐 |
| 典型问题 | 对基金、股票、币、存款等概念理解模糊 |

### 核心痛点

- 容易把理财等同于赚钱。
- 容易忽视本金风险。
- 不理解风险等级。
- 可能用生活费做高风险投资。

### 产品价值

- 用学生语言解释理财概念。
- 判断风险等级。
- 强调生活费流动性和应急金优先。
- 避免具体产品推荐。

# 6. 用户需求分析

## 6.1 需求总览

| 需求方向 | 用户问题 | 产品能力 |
|---|---|---|
| 攒钱规划 | 我这个月能不能攒下目标金额？ | 攒钱规划 Agent |
| 消费管控 | 我今天钱都花在哪了？ | 消费复盘 Agent |
| 风险认知 | 我能不能拿生活费买基金/股票/币？ | 风险教育 Agent |
| 陪伴监督 | 有没有人能提醒我别乱花？ | AI 搭子人设 |
| 低门槛使用 | 我不想学复杂理财软件 | 自然语言对话 |
| 可执行建议 | 我到底明天该怎么做？ | 一步行动建议 |

## 6.2 用户故事

### 用户故事 1：攒钱规划

作为一名有攒钱目标的大学生，  
我希望输入生活费、固定支出和攒钱目标，  
以便系统告诉我这个目标是否可行，以及每天应该控制在多少钱以内。

### 用户故事 2：消费复盘

作为一名经常月末吃紧的大学生，  
我希望用一句话输入今天的消费记录，  
以便系统帮我识别哪些是必要消费，哪些是可以优化的消费。

### 用户故事 3：理财风险教育

作为一名刚开始接触理财的大学生，  
我希望询问基金、股票、虚拟资产等问题，  
以便系统用我能听懂的话解释风险，而不是直接推荐我买什么。

### 用户故事 4：陪伴式提醒

作为一名执行力不稳定的大学生，  
我希望系统像搭子一样提醒我下一步怎么做，  
以便我不会因为建议太复杂而放弃执行。

# 7. 产品范围

## 7.1 MVP 范围

本次比赛版本只实现 3 个核心能力：

1. 攒钱规划。
2. 消费复盘。
3. 入门理财风险教育。

## 7.2 功能优先级

| 功能 | 优先级 | 是否纳入 v1.0 | 说明 |
|---|---|---|---|
| 手机端首页 | P0 | 是 | 扫码后的第一印象 |
| AI 搭子对话页 | P0 | 是 | 智能体主入口 |
| 攒钱规划 | P0 | 是 | 核心能力 |
| 消费复盘 | P0 | 是 | 核心能力 |
| 风险教育 | P0 | 是 | 核心能力 |
| 结果卡片化 | P0 | 是 | 提升展示效果 |
| 本地计划存储 | P1 | 是 | 使用 localStorage |
| 轻量 RAG | P1 | 是 | 增强可解释性和边界 |
| 财务健康分 | P2 | 可选 | 可作为亮点展示 |
| 周报/月报 | P2 | 否 | 后续迭代 |
| 用户登录 | P3 | 否 | MVP 不做 |
| 账户绑定 | P3 | 否 | MVP 不做 |
| 真实金融产品推荐 | 禁止 | 否 | 与非营销型定位冲突 |

## 7.3 不做范围

本版本明确不做：

- 登录注册。
- 实名认证。
- 银行账户绑定。
- 微信/支付宝账单导入。
- 真实记账数据库。
- 投资产品推荐。
- 基金、股票、虚拟资产购买。
- 借贷产品推荐。
- 社交社区。
- 大规模用户系统。
- 后台管理系统。

# 9. 核心用户流程

## 9.1 首次进入流程

```text
用户扫码
→ 打开手机端首页
→ 查看产品定位
→ 选择核心入口
→ 进入对应功能
→ 输入信息
→ 查看 Agent 结果
→ 继续追问或返回首页
```

## 9.2 攒钱规划流程

```text
点击“帮我做攒钱计划”
→ 输入月生活费
→ 输入固定支出
→ 输入攒钱目标
→ 输入剩余天数
→ 点击生成计划
→ 后端计算预算
→ Agent 生成解释
→ 前端展示计划卡片
→ 保存最近计划
```

## 9.3 消费复盘流程

```text
点击“复盘今天消费”
→ 输入自然语言消费记录
→ 后端提取消费项目和金额
→ 分类消费
→ 判断是否超预算
→ 生成复盘建议
→ 前端展示分类卡片
```

## 9.4 风险教育流程

```text
点击“问一个理财问题”
→ 输入问题
→ 后端识别风险类型
→ 检索风险知识库
→ 触发风险边界
→ Agent 生成白话解释
→ 前端展示风险判断卡片
```

# 10. 页面需求

## 10.1 首页

### 页面目标

让用户在 3 秒内理解产品价值，并知道可以做什么。

### 页面元素

| 模块 | 内容 |
|---|---|
| 顶部区域 | 产品名称、欢迎语 |
| 主张文案 | 不推销理财产品，只陪你把生活费管明白 |
| 状态卡 | 今日可花预算、本月攒钱目标、本周提醒 |
| 快捷入口 | 攒钱规划、消费复盘、理财问答 |
| 底部导航 | 首页、搭子、复盘、计划 |

### 首页示例文案

```text
攒钱搭子 AI

不推销理财产品，
只陪你把生活费管明白。

今天想先做什么？
```

### 验收标准

- 页面在手机端展示正常。
- 三个功能入口清晰可点击。
- 首屏不超过一屏半。
- 产品定位表达清楚。

## 10.2 搭子对话页

### 页面目标

承载智能对话体验。

### 页面元素

| 模块 | 内容 |
|---|---|
| 顶部 | AI 搭子头像、在线状态 |
| 聊天流 | 用户消息、Agent 消息 |
| 快捷按钮 | 做攒钱计划、复盘消费、解释风险 |
| 输入框 | 用户自然语言输入 |
| 发送按钮 | 触发 Agent |

### 首条消息

```text
你好，我是你的攒钱搭子。
我不会推销理财产品，只会帮你做三件事：
1. 拆攒钱目标；
2. 复盘今天消费；
3. 解释理财风险。
你今天想先处理哪件事？
```

### 验收标准

- 支持用户输入。
- 支持快捷入口。
- 支持展示 Agent 回复。
- 支持展示结构化卡片。

## 10.3 攒钱计划页

### 页面目标

帮助用户完成预算和攒钱目标拆解。

### 输入字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 月生活费 | number | 是 | 用户每月固定生活费 |
| 兼职收入 | number | 否 | 本月额外收入 |
| 奖学金/其他收入 | number | 否 | 本月额外收入 |
| 固定支出 | number | 是 | 房租、交通、电话费等 |
| 攒钱目标 | number | 是 | 本月希望攒下的金额 |
| 剩余天数 | number | 是 | 本月剩余天数 |
| 控制风格 | enum | 否 | 严格/平衡/宽松 |

### 输出字段

| 字段 | 说明 |
|---|---|
| 目标可行性 | 不可行/压力较大/基本可行/较可行 |
| 本月可消费金额 | 可用于日常消费的金额 |
| 每日预算 | 每日建议消费上限 |
| 每周预算 | 每周建议消费上限 |
| 风险点 | 可能影响目标的消费类型 |
| 行动建议 | 本周可执行建议 |

### 验收标准

- 用户输入完整后可生成结果。
- 预算计算正确。
- 结果以卡片展示。
- 可保存最近计划。

## 10.4 消费复盘页

# 11. 智能体设计

## 11.1 智能体名称

**攒钱搭子**

## 11.2 智能体角色

攒钱搭子是一个面向大学生的 AI 财务陪伴智能体。

它是：

- 同学式陪伴者。
- 预算教练。
- 消费复盘助手。
- 风险教育守门人。

它不是：

- 投资顾问。
- 销售人员。
- 基金推荐助手。
- 银行客户经理。
- 家长式监督者。

## 11.3 智能体任务

| 任务 | 说明 |
|---|---|
| 攒钱规划 | 帮用户拆解目标，生成每日预算 |
| 消费复盘 | 帮用户分类消费，识别可优化项 |
| 风险教育 | 帮用户理解理财风险 |
| 信息补全 | 信息不足时主动追问 |
| 行动建议 | 每次输出一个具体行动 |

## 11.4 智能体边界

智能体必须遵守：

1. 不推荐具体股票、基金、币种。
2. 不推荐具体金融产品。
3. 不承诺收益。
4. 不鼓励借贷投资。
5. 不建议用生活费进行高风险投资。
6. 不制造收益焦虑。
7. 不把理财等同于赚钱。
8. 不编造用户信息。
9. 不在信息不足时强行输出结论。
10. 风险教育类回答必须提示“不构成投资建议”。

## 11.5 智能体系统 Prompt

```text
你是“攒钱搭子 AI”，一名面向大学生的理财陪伴智能体。

你的任务不是销售金融产品，也不是提供投资收益建议，而是帮助大学生完成三类任务：
1. 制定攒钱和预算计划；
2. 复盘日常消费并给出可执行的消费优化建议；
3. 用通俗语言解释入门理财知识和投资风险。

你的语气应当像同学和教练的结合：轻松、具体、克制、不说教。你需要帮助用户建立预算意识、储蓄习惯、应急金意识和风险认知。

你必须遵守以下边界：
- 不推荐具体股票、基金、币种或金融产品；
- 不承诺收益；
- 不鼓励借贷投资；
- 不建议用户使用生活费进行高风险投资；
- 对股票、基金、虚拟资产等问题必须强调风险；
- 对学生用户优先建议建立预算、应急金和风险认知；
- 本回答仅用于理财知识教育，不构成投资建议。

当用户信息不足时，你必须主动追问关键字段，而不是编造。
当你输出建议时，必须结构化，并给出一个具体的下一步行动。
```

# 13. Workflow 1：攒钱规划

## 13.1 输入

```json
{
  "monthlyIncome": 3000,
  "extraIncome": 0,
  "fixedExpense": 1200,
  "savingGoal": 600,
  "remainingDays": 25,
  "style": "balanced"
}
```

## 13.2 处理逻辑

```text
1. 校验 monthlyIncome、fixedExpense、savingGoal、remainingDays。
2. 计算 totalIncome。
3. 计算 availableIncome。
4. 计算 monthlySpendable。
5. 计算 dailyBudget。
6. 计算 weeklyBudget。
7. 判断 feasibility。
8. 检索预算规则知识库。
9. 调用 LLM 生成学生化解释。
10. 返回结构化结果。
```

## 13.3 公式

```text
totalIncome = monthlyIncome + extraIncome
availableIncome = totalIncome - fixedExpense
monthlySpendable = availableIncome - savingGoal
dailyBudget = monthlySpendable / remainingDays
weeklyBudget = dailyBudget * 7
```

## 13.4 可行性规则

| 条件 | 判断 |
|---|---|
| monthlySpendable < 0 | 不可行 |
| dailyBudget < 30 | 压力较大 |
| dailyBudget >= 30 且 <= 60 | 基本可行 |
| dailyBudget > 60 | 较可行 |

## 13.5 返回示例

```json
{
  "type": "saving_plan_result",
  "feasibility": "基本可行",
  "totalIncome": 3000,
  "fixedExpense": 1200,
  "savingGoal": 600,
  "monthlySpendable": 1200,
  "dailyBudget": 48,
  "weeklyBudget": 336,
  "riskPoints": ["外卖", "奶茶", "打车"],
  "nextAction": "本周先把外卖控制在2次以内。",
  "reply": "你的目标基本可行，但需要控制高频小额消费。"
}
```

## 14.3 分类规则

| impulsive | 游戏充值、盲盒、非计划购物 |
| learning | 教材、课程、考试报名、打印资料 |

# 26. 风险与应对

## 26.1 模型回答不稳定

应对：

- 使用结构化 Prompt。
- 要求返回 JSON。
- 关键计算由代码执行。
- 前端按结构化字段渲染。

## 26.2 金融风险表达不当

应对：

- 高风险问题单独 Workflow。
- 系统 Prompt 明确禁止。
- 加免责声明。
- 做高风险测试样例。

## 26.3 Demo 现场失败

应对：

- 提前录制视频。
- 准备固定演示输入。
- 前端提供默认样例按钮。
- 准备截图备份。

## 26.4 范围失控

应对：

- 不做登录。
- 不做账户绑定。
- 不做真实金融产品。
- 只做 3 个核心能力。

## 28.2 最终总结

攒钱搭子 AI 的核心不是帮助大学生追逐高收益，而是帮助他们从第一笔生活费开始，学会预算、储蓄、复盘和理解风险。

最终产品表达：

> 攒钱搭子 AI 是一个面向大学生的财务习惯养成 Agent。它通过攒钱规划、消费复盘和风险教育，帮助学生从“月末没钱”走向“有计划地用钱”，从“盲目跟风理财”走向“理解风险再行动”。
