## P2｜赛题理解

### 市场研究
- 中国在校大学生超 4700 万，人均月生活费 1500-3500 元，自主支配权持续提升
- 月生活费 + 奖学金 + 兼职收入构成多元现金流，但缺乏管理工具
- 现有工具：专业理财 App 偏向投资者，功能复杂；记账 App 依赖手动输入，留存率低；AI 对话机器人缺乏金融场景深度

### 目标场景
大学生在日常生活中面临的 4 类高频财务决策时刻：
1. **花钱前**：要不要买这件东西？（冲动消费场景）
2. **花钱后**：今天超支了，怎么补？（复盘修复场景）
3. **月末时**：还剩 300 元撑 8 天，怎么办？（急救场景）
4. **理财前**：能不能拿这笔钱买基金？（风险判断场景）

### 核心问题
| 问题 | 现象 | 后果 |
|------|------|------|
| 没有预算感知 | 月初随意花，月末吃土 | 被动节衣缩食 |
| 情绪性消费 | 压力大时更容易冲动购买 | 超支、后悔循环 |
| 盲目跟风投资 | 同学赚了就跟进 | 生活费进入高风险资产 |
| 超支后放弃 | "这个月废了，下个月重来" | 计划永远重启，永远失败 |

### 为什么值得做
- **时机窗口**：大学是财务行为习惯形成的关键期，此阶段干预效果最显著
- **差异化空间**：市场无轻量化、陪伴式、非营销型产品
- **招行价值**：培育财务认知健全的年轻客群，建立信任关系先于产品关系

## P3｜用户需求分析

### 用户画像
**典型用户：小林，大二学生**
- 月生活费 3000 元（家庭汇款）+ 兼职 500 元
- 固定支出：房租 800 + 话费 50 = 850 元
- 剩余可支配 2650 元，但每月末经常余额不足 200 元
- 有攒钱意愿（想存 600/月），但执行率不足 30%
- 频繁点外卖、买奶茶，事后有轻微自责感
- 看到同学聊基金/股票时感到焦虑，不知该不该跟

### 行为特征
- **月初宽松，月末急救**：开头两周随意，最后一周拼命省
- **记账打开率低**：下载记账 App → 用 3 天 → 放弃
- **情绪驱动消费**：压力大 → 刷外卖 / 买零食 → 短暂舒适 → 自责
- **跟风但不理解**：听到"基金"就想参与，但不懂风险等级
- **超支后全盘放弃**：一次超支就觉得"这个月完了"

### 核心痛点（按优先级）
1. **痛点 1（最高）**：没有实时预算感——不知道今天还能花多少
2. **痛点 2**：超支后不知道怎么恢复——容易放弃整个计划
3. **痛点 3**：月末资金紧张时无处求助——只能硬扛
4. **痛点 4**：想了解理财但怕被骗或亏掉生活费——缺乏可信的入门引导

### 需求优先级矩阵
| 需求 | 频率 | 紧迫度 | 优先级 |
|------|------|--------|--------|
| 每日预算提醒与复盘 | 每天 | 高 | ★★★★★ |
| 超支后修复计划 | 每周 | 高 | ★★★★★ |
| 月末急救预算 | 每月1-2次 | 极高 | ★★★★☆ |
| 冲动消费拦截 | 每周 | 中 | ★★★★☆ |
| 入门投资引导 | 偶发 | 中 | ★★★☆☆ |
| 历史消费分析 | 每周 | 低 | ★★★☆☆ |

## P4｜用户旅程 / 业务流程

### 全生命周期流程

```
【触发】用户发送消息
        ↓
【安全过滤 Step 0】输入合规检查
  ├─ 拦截：推荐股票/承诺收益/借钱投资 → 返回合规安全回复
  └─ 通过 ↓
【意图识别 Step 1】11类 Intent 精准路由
  打卡 / 月末急救 / What-if推演 / 冲动消费 / 消费替代
  情绪陪伴（优先级最高）/ 消费复盘 / 攒钱规划 / 风险教育
  投资准备度 / 闲聊跟进
        ↓
【情绪感知 Step 2】5 类情绪 × 3 级强度
  焦虑 / 挫败 / 自责 / 经济压力 / 比较焦虑
  → 影响回复温度和情绪承接策略
        ↓
【知识检索 Step 3】RAG 向量检索
  71 条领域知识 × 标签+关键词双重匹配
        ↓
【工作流计算 Step 4】确定性计算
  槽位提取（月收入/固定支出/目标）
  数据库读取（历史消费/用户档案）
  算法计算（预算/可行性/修复方案）
  结果写库（消费记录/档案更新）
        ↓
【提示词组装 Step 5】分层 Prompt
  身份定义 + 数据库上下文 + RAG知识 + 工作流结果 + 格式要求
        ↓
【LLM 调用 Step 6】DeepSeek API
  temperature 随意图×情绪动态调整 / 携带10轮历史上下文
        ↓
【输出过滤 Step 7】禁止词检查
  拦截：稳赚/推荐买入/必赚/翻倍 → 替换为合规回复
        ↓
【响应组装 Step 8】结构化返回
  reply（主文本）+ card（可视化卡片）+ nextSuggestions + syncNote
        ↓
【前端渲染】数据同步
  聊天气泡 + 功能卡片 + 数据面板实时刷新（mb-data-updated 事件）
```

### 典型场景路径示例
**场景：用户说"今天食堂18，奶茶16，外卖35，打车22"**
1. Intent → `expense_review`
2. 解析消费条目 → 分类（必要/可优化/冲动/学习）
3. 对比每日预算 → 计算是否超支
4. 超支则计算修复方案（剩余天数平摊）
5. 自动写入数据库 today 记录（若已有则合并）
6. 触发 `mb-data-updated` → 首页数据面板实时刷新
7. 返回：消费复盘卡 + 修复提示 + 明日行动建议

## P5｜产品定位

### 目标用户
在校大学生（18-25 岁），有基本消费能力但缺乏财务规划习惯，使用智能手机频率极高

### 价值主张
**"不是让你存更多，而是帮你做更好的决定"**

招小财不记录过去，而是在财务决策的关键时刻主动介入：
- 花之前：冷静一下，真的需要吗？
- 花之后：超了也没关系，我们来修
- 月末了：先算清楚，再慌不迟
- 投资前：数据说话，现在是否合适？

### 差异化对比
| 维度 | 传统记账App | 通用AI助手 | **招小财** |
|------|------------|-----------|----------|
| 核心逻辑 | 记录过去 | 回答问题 | **在关键时刻干预决策** |
| 风险管控 | 无 | 无护栏 | **双向安全边界** |
| 情绪感知 | 无 | 有限 | **5类情绪×3级强度路由** |
| 数据联动 | 手动 | 无DB | **实时写库+前端刷新** |
| 投资引导 | 无 | 泛泛推荐 | **基于用户数据的准备度评估** |
| 用户定位 | 所有人 | 所有人 | **在校大学生专属场景** |

### 核心指标（设计时对标）
- 意图识别准确率 ≥ 90%（11类 intent 覆盖日常场景）
- 安全边界零穿透（双向 guardrail 拦截违规输出）
- DB 数据同步延迟 < 1s（消费记录实时写入）
- 回复字数控制（主回复 ≤ 150 字，符合移动端阅读习惯）

## P7｜核心能力 1：消费行为干预引擎

### 功能逻辑
用户发送消费记录（如"食堂 18，奶茶 16，外卖 35"）→ 系统自动：
1. **解析条目**：正则提取数字+关键词，识别 11 类高频消费场景
2. **四分类**：必要消费 / 可优化消费 / 冲动消费 / 学习投资
3. **对比预算**：计算当日总额 vs 每日预算
4. **区间判断**：绿色（≤90%）/ 黄色（90-125%）/ 红色（>125%）
5. **修复计算**（超支时）：超支金额 ÷ 剩余天数 = 每天需少花 X 元
6. **自动写库**：合并当天记录，触发前端实时刷新

**冲动消费拦截**（并行子流程）
- 用户说"想买个 399 的耳机" → 识别为 `impulse_check`
- 计算：399 = X 天预算，占攒钱目标 Y%
- 推荐：冷静 24 小时 / 列入计划 / 可接受
- 不说"不能买"，而是"先等一等，明天还想买就买"

### AI 能力
- LLM temperature = 0.3（事实导向，避免过度安慰）
- 情绪强时（自责/挫败）temperature → 0.45，先情绪承接再说数字
- Prompt 强制要求：超支场景必须包含具体修复方案（不能只说"注意"）
- 系统词禁止："你又超支了" / "你必须控制" / "你的消费习惯很差"

### 关键界面
- **消费复盘卡**：总支出 + 绿/黄/红区域标记 + 分类占比
- **修复提示条**：平摊 N 天，每天少花 ¥X（显示在卡片副标题）
- **明日行动**：一句话具体行动（如"明天外卖换食堂"）

### 体验路径
```
用户发"食堂18奶茶16外卖35"
→ 1.2s 内返回分类卡片
→ 绿色区：鼓励 + 可优化提示
→ 红色区：先承接情绪 → 修复方案 → 一件小事
→ 消费同时写入数据库，首页面板自动更新
```

## P8｜核心能力 2：攒钱规划 + 月末急救

### 功能逻辑

**攒钱规划（Saving Plan）**
- 提取三元素：月生活费、固定支出、攒钱目标
- 若信息不全 → 精准追问（不是"请填写所有字段"，而是"还差一个数字：固定支出多少？"）
- 计算链：总收入 → 扣固定支出 → 扣攒钱目标 → 本月可花 → 每日预算 → 每周预算
- 可行性评级：不可行 / 压力较大 / 基本可行 / 较可行
- **增量更新**：说"奖学金发了 500" → 系统识别 delta，extraIncome += 500，重算预算，同步更新

**月末急救（Rescue Mode）**
- 触发：说"还剩 300 块，要撑 8 天"
- 计算：每日存活上限 = 余额 ÷ 天数
- 风险等级：< 20 元/天（非常紧张）/ 20-40（偏紧张）/ > 40（尚可控）
- 输出：必须暂停的消费清单 + 食物预算 + 今天一件能做到的事
- 语气：先安抚，再给数字，不说"你怎么花成这样"

### AI 能力
- Slot 提取支持绝对值 + 增量值双模式
- 槽位缺失时生成精准追问（区分差 1 个 vs 差 3 个字段）
- 月末急救 temperature = 0.5（实用为主，带情绪温度）
- Prompt 要求：急救方案必须包含今天就能执行的 1 件事

### 关键界面
- **每日预算大卡**：64px 数字 + 可行性徽章 + 每周/本月可花/目标三格
- **月末急救卡**：每日上限 + 风险等级颜色 + 暂停清单 + 今日行动
- **首页数据面板**：实时同步数据库，`mb-data-updated` 事件驱动刷新

### 体验路径
```
用户打开攒钱计划页面
→ 自动从数据库拉取最新档案（syncFromDB）
→ 填写/修改三元素
→ 提交 → 后台计算 + 写库
→ 首页面板同步更新每日预算数字
OR
用户说"还剩200块要撑5天"
→ 识别为月末急救 → 计算每日40元上限
→ 推送今日急救方案
```

## P9｜核心能力 3：基于数据的投资准备度评估

### 为什么这个能力最能体现差异化
通用 AI 说"基金有风险，要谨慎"——这没有用。  
招小财会先算你的账：**你的余额撑不撑得到月底，才是是否适合投资的第一个问题。**

### 功能逻辑

**触发场景（4 类）**
1. FOMO 跟风："我同学买基金赚了，我是不是也该买？"
2. 具体金额："我能不能拿 500 块买基金试试？"
3. 阶段询问："我现在适合做什么理财？"
4. 省钱后投资："省一点能不能买基金？"

**数据计算流程**
```
读取数据库 → 本月已花 / 月可支配额
计算余额 = 月可支配 - 已花
近7天日均 → 预计到月底需要 = 日均 × 剩余天数
现金流判断 = 余额 vs 预计支出
应急金判断 = 已存金额 vs 月收入×0.5
投资比例 = 提及金额 / 余额（> 30% → 危险）
综合评级：理财准备期 / 过渡期 / 可以考虑入门
```

**5段固定回复结构**
1. 先给结论（适不适合真实投入）
2. 引用用户数据（具体数字说话）
3. 风险判断（这笔钱是不是生活费，有无应急金）
4. 替代路径（四个钱袋：生活→应急→目标→学习）
5. 边界声明（不构成投资建议）

**安全三问**（投资前必问）
- Q1：这笔钱是不是生活费或近期要用的钱？
- Q2：有没有单独的应急金（≥ 300-500 元）？
- Q3：短期亏损 5-10% 会不会影响吃饭交通学习？

### AI 能力
- LLM temperature = 0.35（事实+温度兼顾）
- 禁止输出："可以买一点低风险基金"（泛泛话术）
- 必须输出：引用用户余额、日均、预计支出的具体数字
- RAG 动态加载：4 个钱袋框架 + 安全三问 + 货币基金入门
- FOMO 场景特殊处理：先承接"这种感觉很正常"，再用数字说明不适合

### 关键界面
**投资准备度卡片**：
- 三格数据（当前余额 / 预计支出 / 日均消费）
- 准备度标签（暂不建议 / 谨慎观望 / 可以学习入门）+ 颜色编码
- 应急金状态标记
- 4 条编号行动建议
- 免责声明（固定底部显示）

## P10｜智能体设计

### 任务拆解（11 类 Intent 覆盖）
| 类别 | Intent | 核心任务 |
|------|--------|---------|
| 消费干预 | expense_review | 分类 + 对比 + 修复计划 |
| 消费干预 | impulse_check | 金额换算 + 冷静期建议 |
| 消费干预 | spending_alternative | 4种替代方案不禁止 |
| 攒钱规划 | saving_plan | 槽位提取 + 预算计算 |
| 急救模式 | rescue_mode | 存活预算 + 停止清单 |
| 场景推演 | what_if | 行为改变量化影响 |
| 投资教育 | investment_readiness | 数据判断准备度 |
| 投资教育 | risk_education | 风险等级 + 安全三问 |
| 情绪支持 | emotional_support | 承接优先，行动轻 |
| 习惯强化 | checkin | 打卡正向反馈 |
| 消费分析 | spending_analysis | 历史复盘 + 趋势洞察 |

### 工具调用（Tool Use）
- **DB 读取**：每次对话读取用户档案 + 最近14天消费记录
- **DB 写入**：消费复盘自动写库，增量更新自动合并
- **预算计算器**：deterministic 算法（不依赖 LLM 算数）
- **RAG 检索**：标签+关键词双路匹配，动态注入 Prompt
- **Slot 解析器**：绝对值 + 增量值双模式提取，支持"多加500奖学金"类自然语言

### 记忆 / 知识库
**短期记忆**：
- 每次请求携带最近 10 轮对话历史（真实 message 数组，非文本拼接）
- 当次请求 DB Profile + 当日消费记录注入 Prompt

**长期知识库**：
- 71 条领域知识，分 16 类标签
- 覆盖：预算原则、消费心理、风险教育、情绪支持、投资框架
- 检索策略：按 intent 预设标签组 → 关键词匹配打分 → top-k 注入 Prompt

**用户档案持久化**：
- profile：月收入/固定支出/攒钱目标/每日预算/已存金额/连击天数
- spending：每日消费记录，含分类合计和超支信息

### 异常兜底
| 异常类型 | 处理策略 |
|---------|---------|
| 输入含违规词（推荐股票/稳赚/借钱投资）| Step 0 拦截，返回固定合规回复 |
| LLM 输出含违规短语（翻倍/必赚/推荐买入）| Step 7 检测并替换为合规回复 |
| LLM 返回非 JSON 格式 | parseJSON 容错解析，回退 fallback 文案 |
| 消息金额无法解析 | 返回 needsMoreInfo，用友好追问引导补充 |
| API 调用失败 | Demo 模式 fallback，返回确定性回复不报错 |
| DB 无用户数据 | 使用预设默认档案，提示用户先做攒钱计划 |

### 典型失败案例与优化
**失败案例 1**：
- 输入："食堂18奶茶16打车22"（无"元"字）
- 问题：早期版本 EXPENSE_RE 要求有"元"字才触发

**失败案例 2**：

**失败案例 3**：
- 问题：聊两句后 LLM 不记得上文

- 优化：改为 OpenAI 标准 messages 数组格式，携带真实多轮对话
