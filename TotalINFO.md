# 1. 项目总览

## 1.1 产品名称

**攒钱搭子 AI**

## 1.2 一句话定位

**面向大学生的轻量化、陪伴式、非营销型理财陪伴智能体，帮助学生完成攒钱规划、消费复盘和入门理财风险判断。**

## 1.3 产品主张

> 不推销理财产品，只陪你把生活费管明白。

## 1.4 产品形态

本项目应以**手机端可扫码体验 Demo** 为核心交付形态，而不是 PC 端后台或纯 PPT 概念方案。

推荐形态：

```text
手机端 H5 / PWA
+ AI 对话机器人主入口
+ 场景化快捷入口
+ 结果卡片化展示
+ Workflow Agent 后端
+ 轻量 RAG 知识库
+ DigitalOcean 云服务器部署
+ 可体验二维码
```

## 1.5 核心交付目标

本项目最终需要交付：

1. **可体验 Demo**
   - 手机扫码即可打开。
   - 支持 2–3 项核心能力。
   - 能稳定完成演示路径。

2. **产品方案 PPT**
   - 12 页以内。
   - 包含用户需求分析、产品定位、智能体设计思路、核心界面、评测方式和评测结果。
   - 若使用 AI 生成内容，相关页面右上角标注红色加粗“AI辅助”。

3. **3 分钟以内横屏演示视频**
   - 展示智能体人设。
   - 展示交互逻辑。
   - 展示服务流程。
   - 展示核心功能模块。
   - 本人无需出镜。

4. **可选附件**
   - 流程图。
   - 思维导图。
   - Workflow 图。
   - 测试样例。
   - 可打包为 ZIP 提交。

- 生活费自主支配
- 兼职收入
- 奖学金
- 长期攒钱规划
- 应急储蓄意识
- 理财认知薄弱
- 盲目跟风
- 忽视投资风险
- 理财搭子
- 轻量化
- 陪伴式
- 非营销型
- 智能对话机器人
- 2–3 项核心能力
- 可体验二维码
- 智能体设计
- 功能实现
- 应用演示
- 评测方式和评测结果

## 2.2 产品经理视角下的核心命题

这个赛题不是在考察“能不能做一个聊天机器人”，而是在考察参赛者是否能完成以下产品判断：

1. **能否准确识别目标用户**
   - 大学生不是成熟投资者。
   - 大学生真实问题不是“缺少高收益产品”，而是“缺少财务行为管理”。

2. **能否把抽象金融能力转化为学生场景**
   - 预算规划要变成“今天还能花多少钱”。
   - 风险教育要变成“生活费能不能拿去买基金/股票/币”。
   - 财务规划要变成“我怎么攒下买电脑/旅行/考试报名的钱”。

3. **能否设计真正的 Agent，而非套壳大模型**
   - Agent 要有任务边界。
   - Agent 要有主动追问。
   - Agent 要有 Workflow。
   - Agent 要有风险边界。
   - Agent 要有稳定输出格式。

4. **能否形成可演示闭环**
   - 扫码可体验。
   - 视频可展示。
   - PPT 可解释。
   - 评测可证明。

## 2.3 项目不应该做成什么

本项目不应该做成：

| 错误方向 | 问题 |
|---|---|
| 理财产品推荐机器人 | 容易偏销售，不符合非营销型定位 |
| 泛问答 Chatbot | 与直接问基础模型差异不明显 |
| 记账 App | 只能记录，缺少智能体陪伴和行动建议 |
| 专业投资分析工具 | 不适合大学生用户和赛题语境 |
| 银行营销入口 | 与“轻量化、陪伴式、非营销型”冲突 |
| 复杂全功能金融 App | 开发成本高，Demo 展示不聚焦 |

## 2.4 项目应该做成什么

本项目应该做成：

> 一个手机端 AI 理财搭子，以自然语言对话为主入口，通过攒钱规划、消费复盘、入门理财风险教育三个高频场景，帮助大学生建立财务自控系统。

# 3. 背景与市场机会

## 3.1 大学生理财需求变化

随着生活费自主支配权提升，大学生逐渐成为理财入门群体。

大学生常见资金来源包括：

- 家庭生活费。
- 兼职收入。
- 奖学金。
- 实习收入。
- 比赛奖金。
- 临时补贴。
- 二手交易收入。

但这些资金通常具有以下特点：

1. **金额不大但频率稳定**
   - 月生活费往往固定。
   - 兼职或奖学金具有阶段性。

2. **资金管理意识较弱**
   - 大多数学生没有系统预算习惯。
   - 消费支出碎片化。

3. **目标支出明显**
   - 电脑、手机、旅行、考试、考证、课程、实习通勤等目标支出普遍存在。

4. **理财认知处于初级阶段**
   - 容易把理财等同于赚钱。
   - 对本金风险、流动性风险、波动风险理解不足。

## 3.2 现有产品缺口

现有理财类产品通常存在以下问题：

| 产品类型 | 问题 |
|---|---|
| 传统银行 App | 功能多但入口重，不够学生化 |
| 基金/券商 App | 偏投资交易，风险门槛高 |
| 记账 App | 记录为主，缺少陪伴和解释 |
| 消费分期/信贷产品 | 可能诱导过度消费，不适合作为正向理财启蒙 |
| 通用大模型 | 能回答问题，但缺少固定流程、边界和产品化体验 |

因此，大学生需要的不是一个“更复杂的金融工具”，而是一个：

> 低门槛、对话式、陪伴式、能帮助形成习惯的理财辅助工具。

## 3.3 金融机构视角的长期价值

虽然本项目不应设计成销售型产品，但从金融机构视角看，年轻用户具有长期价值。

大学生未来可能成为：

- 工资卡用户。
- 储蓄用户。
- 信用卡用户。
- 低风险理财用户。
- 保险用户。
- 贷款用户。
- 长期财富管理客户。

因此，该产品的战略意义不是短期转化，而是：

> 通过非营销型陪伴服务，在年轻用户阶段建立金融信任、账户习惯和风险认知。

这个观点可以作为 PPT 中背景/市场机会页的上层洞察，但不应喧宾夺主。项目主体仍应围绕大学生自身需求展开。

# 4. 用户研究与需求洞察

## 4.1 目标用户定义

核心用户：

> 在校大学生，拥有生活费、兼职收入、奖学金等可支配资金，但缺少预算规划、消费复盘和理财风险认知。

## 4.2 用户细分

### 用户类型一：月末吃紧型

**特征：**

- 月生活费固定。
- 月初消费随意。
- 月末明显压缩开支。
- 不知道钱花在哪里。

**痛点：**

- 缺少每日预算。
- 缺少消费提醒。
- 缺少复盘习惯。

**对应能力：**

- 消费复盘。
- 每日预算。
- 可优化消费识别。

### 用户类型二：目标攒钱型

**特征：**

- 有明确攒钱目标。
- 想买电脑、旅行、考试、考证、课程等。
- 缺少目标拆解能力。

**痛点：**

- 不知道目标是否现实。
- 不知道每周/每天该花多少。
- 容易被临时消费打断。

**对应能力：**

- 攒钱规划。
- 目标可行性判断。
- 周度预算拆解。

### 用户类型三：理财小白型

**特征：**

- 刚接触基金、股票、虚拟资产等概念。
- 经常刷到高收益内容。
- 对风险理解不充分。

**痛点：**

- 容易盲目跟风。
- 忽视本金风险。
- 可能用生活费进行高风险投资。

**对应能力：**

- 入门理财风险教育。
- 高风险问题识别。
- 非投资建议提示。

### 用户类型四：需要监督型

**特征：**

- 知道要省钱，但缺少执行力。
- 希望有人提醒但不想被说教。
- 更接受同伴式陪伴。

**痛点：**

- 缺少持续反馈。
- 缺少低压力提醒。
- 缺少复盘机制。

**对应能力：**

- AI 搭子人设。
- 每周复盘。
- 一步行动建议。

## 4.3 JTBD 分析

### Job 1：当我月初拿到生活费时

我想知道这个月每天最多能花多少钱，  
以便我不会月末突然没钱。

### Job 2：当我想攒钱买某个东西时

我想知道目标是否现实、每周应该控制多少钱，  
以便我能坚持完成目标。

### Job 3：当我一天花了很多小钱时

我想知道哪些消费是必要的，哪些是可以优化的，  
以便我明天可以调整。

### Job 4：当我看到别人推荐基金、股票或虚拟资产时

我想知道自己是否适合参与、风险是什么，  
以便我不会盲目跟风。

### Job 5：当我想开始理财但不懂专业术语时

我想用简单的话理解基本概念，  
以便我先建立风险意识，而不是直接买产品。

## 4.4 用户核心需求总结

用户需求可以归纳为三个层级：

```text
第一层：管住今天的钱
- 今日预算
- 消费复盘
- 可优化消费识别

第二层：攒下未来的钱
- 攒钱目标
- 月度预算
- 周度拆解
- 应急储蓄

第三层：看懂理财的风险
- 入门知识解释
- 风险等级理解
- 不盲目跟风
- 不借贷投资
```

这三个层级对应产品的三条核心 Workflow：

1. 消费复盘。
2. 攒钱规划。
3. 风险教育。

# 5. 产品定位与价值主张

## 5.1 产品定位

**攒钱搭子 AI 是一个面向大学生的手机端 AI 理财陪伴智能体。它不做具体投资推荐，而是帮助学生建立预算、储蓄、复盘和风险认知。**

### 对用户的价值

| 用户问题 | 产品价值 |

# 7. 产品差异化与核心亮点

## 7.1 差异化一：从“理财产品”转向“财务习惯”

很多理财工具默认用户已经有投资需求，但大学生真正需要的是：

- 预算能力。
- 储蓄意识。
- 消费复盘。
- 风险认知。

本产品将理财启蒙的起点从“买什么”改为“怎么管好自己的钱”。

## 7.2 差异化二：AI 搭子，而非 AI 理财经理

AI 理财经理容易产生销售感和距离感。  
AI 搭子更适合学生：

- 低压力。
- 更日常。
- 更容易持续使用。
- 更适合陪伴和提醒。

## 7.3 差异化三：Workflow Agent，而非普通 Chatbot

普通 Chatbot 只是回答问题。  
本产品的 Agent 会：

- 判断用户意图。
- 主动追问信息。
- 调用计算规则。
- 检索知识库。
- 执行风险检查。
- 输出结构化结果。
- 给出下一步行动。

## 7.4 差异化四：结果卡片化

为了增强 Demo 说服力，Agent 输出不只是文字，而是转化为：

- 预算结果卡。
- 消费分类卡。
- 风险判断卡。
- 明日行动卡。
- 目标进度卡。

这让手机端界面更像真实产品。

## 7.5 差异化五：可评测、可解释

本项目提供评测方案：

- 预算计算是否正确。
- 消费分类是否合理。
- 高风险问题是否被正确处理。
- 输出是否可理解。
- 是否能完成用户任务。

这能体现产品经理对智能体质量控制的理解。

## 7.6 创新亮点建议

### 亮点 1：三层成长模型

```text
管住今天的钱
→ 攒下未来的钱
→ 看懂理财的风险
```

### 亮点 2：学生财务健康分

可以在 Demo 中加入一个简单评分概念：

```text
财务健康分 = 预算执行 + 储蓄目标 + 消费结构 + 风险认知
```

MVP 不需要复杂实现，但可以在 PPT 中作为未来迭代或轻量展示。

### 亮点 3：一句行动建议

每次回复最后都给出：

```text
今天只做一件事：
明天只改一件事：
本周只坚持一个目标：
```

这比泛泛建议更容易行动。

### 亮点 4：反营销风险守门人

对“能不能买币”“能不能借钱投资”等问题，Agent 不仅回答，还体现“风险守门人”能力。

### 亮点 5：场景化快捷入口

首页不让用户思考“我要问什么”，而是直接给三个入口：

- 帮我做攒钱计划。
- 复盘今天消费。
- 问一个理财问题。

这降低使用门槛。

# 8. 用户旅程与服务蓝图

## 8.1 用户旅程

### 阶段一：进入产品

用户扫码进入手机端页面。

用户心理：

- 这是什么？
- 能帮我做什么？
- 会不会推销理财产品？

产品回应：

- 首屏明确说明“不推销理财产品”。
- 三个入口清晰展示。

# 12. 智能体人设设计

## 12.1 人设定义

**攒钱搭子 AI** 是一个陪伴大学生进行财务习惯养成的 AI 搭子。

它不是：

- 银行销售。
- 投资顾问。
- 基金推荐助手。
- 记账工具。
- 家长式监督者。

它是：

- 同学式陪伴者。
- 预算教练。
- 消费复盘助手。
- 风险教育守门人。

## 12.2 人设三层能力

```text
像同学一样陪伴：
- 语气轻松。
- 不制造压力。
- 理解学生消费场景。

像教练一样提醒：
- 给出预算。
- 指出问题。
- 让建议可执行。

像风控一样守边界：
- 不荐股。
- 不荐基。
- 不荐币。
- 不承诺收益。
- 不鼓励借贷投资。
```

## 12.3 对话风格原则

应该：

- 直接。
- 清楚。
- 学生化。
- 少术语。
- 多行动建议。
- 信息不足时追问。

不应该：

- 夸大收益。
- 制造焦虑。
- 推销产品。
- 说教。
- 输出长篇理论。
- 编造用户数据。

## 12.4 系统 Prompt

```text
你是“攒钱搭子 AI”，一名面向大学生的理财陪伴智能体。

你的任务不是销售金融产品，也不是提供投资收益建议，而是帮助大学生完成三类任务：
1. 制定攒钱和预算计划；
2. 复盘日常消费并给出可执行的消费优化建议；
3. 用通俗语言解释入门理财知识和投资风险。

你的语气应当像同学和教练的结合：轻松、具体、克制、不说教。你需要帮助用户建立预算意识、储蓄习惯、应急金意识和风险认知。

你必须遵守以下边界：
- 不推荐具体股票、基金、币种或金融产品；
- 不承诺收益；
- 不鼓励借贷投资；
- 不建议用户使用生活费进行高风险投资；
- 对股票、基金、虚拟资产等问题必须强调风险；
- 对学生用户优先建议建立预算、应急金和风险认知；
- 本回答仅用于理财知识教育，不构成投资建议。

当用户信息不足时，你必须主动追问关键字段，而不是编造。
当你输出建议时，必须结构化，并给出一个具体的下一步行动。
```

# 13. Agent Workflow 总体设计

## 13.1 为什么需要 Workflow

如果只是把用户输入直接发给基础模型，这个产品与通用 Chatbot 没有本质区别。

Workflow 的价值是：

- 固定任务边界。
- 提高输出稳定性。
- 支持主动追问。
- 保证预算计算正确。
- 保证风险回答合规。
- 让 Agent 具备产品化能力。

## 13.2 总体 Workflow

```text
用户输入
→ 前端传入 mode 或后端识别意图
→ 信息完整性检查
→ 分发到对应 Workflow
→ 执行规则计算/消费分类/风险判断
→ RAG 检索相关知识
→ 构造 Prompt
→ 调用 LLM
→ 解析结构化结果
→ 前端展示卡片
→ 给出下一步行动
```

## 13.3 意图类型

| 意图 | mode | 说明 |
|---|---|---|
| 攒钱规划 | saving_plan | 处理生活费、支出、目标 |
| 消费复盘 | expense_review | 处理消费文本 |
| 风险教育 | risk_education | 处理理财问题 |
| 继续对话 | follow_up | 对上一次结果追问 |

## 13.4 信息缺失处理

### 攒钱规划缺字段

如果缺少固定支出：

```text
我还需要知道你的固定支出，比如房租、交通、电话费、饭卡等。大概一个月多少？
```

如果缺少攒钱目标：

```text
你这个月想攒多少钱？我会根据目标帮你拆成每日预算。
```

### 消费复盘缺金额

```text
我能识别出你提到的消费项目，但有些金额不完整。可以补充一下每项大概花了多少吗？
```

### 风险教育问题过宽

```text
你是想了解概念，还是想判断自己现在适不适合买？如果是后者，我会优先从生活费和风险承受能力角度判断。
```

# 14. 核心 Workflow 1：攒钱规划

## 14.1 用户输入

```json
{
  "monthlyIncome": 3000,
  "extraIncome": 0,
  "fixedExpense": 1200,
  "savingGoal": 600,
  "remainingDays": 25,
  "style": "balanced"
}
```

## 14.2 处理步骤

```text
1. 检查字段完整性。
2. 计算总收入。
3. 计算可支配金额。
4. 计算本月可消费金额。
5. 计算每日预算。
6. 计算每周预算。
7. 判断目标可行性。
8. 检索预算原则和应急金原则。
9. 生成结构化建议。
10. 存储最近一次计划到 localStorage。
```

## 14.3 计算公式

```text
总收入 = 月生活费 + 兼职收入 + 奖学金等额外收入

可支配金额 = 总收入 - 固定支出

本月可消费金额 = 可支配金额 - 攒钱目标

每日预算 = 本月可消费金额 / 剩余天数

每周预算 = 每日预算 × 7
```

## 14.4 可行性判断

| 条件 | 判断 |
|---|---|
| 本月可消费金额 < 0 | 不可行 |
| 每日预算 < 30 | 压力较大 |
| 每日预算 30–60 | 基本可行 |
| 每日预算 > 60 | 较可行 |

## 14.5 输出格式

```json
{
  "type": "saving_plan_result",
  "feasibility": "基本可行",
  "totalIncome": 3000,
  "fixedExpense": 1200,
  "savingGoal": 600,
  "monthlySpendable": 1200,
  "dailyBudget": 48,
  "weeklyBudget": 336,
  "riskPoints": ["外卖", "奶茶", "打车"],
  "nextAction": "本周先把外卖控制在2次以内。",
  "reply": "你的目标基本可行，但需要控制高频小额消费。"
}
```

```json
    "content": "回答应避免销售导向，避免使用稳赚、必买、高收益等表达。建议以教育、提醒、陪伴和行动建议为主。"
  }
]
```

## 17.4 检索方式

MVP 采用标签 + 关键词检索：

```text
根据 mode 确定候选 tags
→ 根据用户输入关键词匹配知识块
→ 返回 2–3 条内容
→ 拼接进入 Prompt
```

后续可升级：

- 向量检索。
- 混合检索。
- 用户长期记忆。

# 18. 风险边界与合规控制

## 18.1 核心原则

本项目必须强调：

- 非营销型。
- 非投资建议。
- 非收益承诺。
- 非具体产品推荐。
- 非借贷投资引导。

## 18.2 禁止输出

Agent 不得输出：

```text
你应该买某某基金。
这只股票值得入。
这个币可以长期持有。
保证收益。
稳赚。
借钱投资也可以。
生活费可以大部分拿去投资。
```

## 18.3 高风险问题处理

用户问：

```text
我可以买币吗？
```

Agent 应答方向：

```text
不建议你用生活费参与虚拟资产投资。虚拟资产价格波动大，且可能带来本金损失风险。对大学生来说，更优先的是保证基本生活、建立应急金和学习风险知识。
```

## 18.4 免责声明

风险教育类回答必须带：

```text
本回答仅用于理财知识教育，不构成投资建议。
```

应对：

- 固定演示路径。
- 准备预设输入。

- 输出 JSON。

- 禁止推荐具体产品。
- 风险问题单独 Workflow。
- 高风险问题强制免责声明。
- 评测中加入高风险样例。

## 25.4 项目范围失控

应对：

- 不做登录。
- 不做账户绑定。
- 不做真实投资。
- 不做复杂数据库。
- 聚焦 3 个核心能力。
